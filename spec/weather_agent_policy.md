1. CheckHourlyForecast intent → hourly 예보(weekly가 포함되지 않은 모든 시간 발화
2. CheckDailyForecast intent → today 예보(현재, 오늘 daily 발화
    1. intent에 따라 today만 응답함, now고 뭐고 걍 today응답 나감

    1. 시간을 물어봤어도 intent에 따라 잘라서 daily data로만 응답

# 2. 날씨 카드 데이터

CheckDailyForecast → todayCard

todayCard 출력 데이터

- 강수확률: 가장 인접한 hourly item의 강수확률
- 지금 온도, 오늘의 최고 온도, 오늘의 최저 온도

CheckWeeklyForecast → todayCard, weeklyCard

weeklyCard 출력 데이터 (사용자 발화 시점에 따라 달라짐)

- 일 단위 동작
    - 특정 일자 요청 (17일 날씨, 금요일 날씨, 내일 날씨)
        - 특정 일자부터 7일
        
- 주 단위 동작 (주말, 주중, 한 주 통으로)
    - 이번주: 내일부터 7일
        - 이번 주 주말
        - 이번 주 주중
        - 이번 주
    - 다음주: 다음 월요일부터 7일
        - 다음 주 주말
        - 다음 주 주중
        - 다음 주
    - 다다음주: 다다음 월요일부터 7일
        - 다다음 주 주말
        - 다다음 주 주중
        - 다다음 주

CheckHourlyForecast → todayCard, hourlyCard

 hourlyCard 출력 데이터

- 사용자가 요청한 시간에 관계없이 현지 기준의 현재시간부터 3시간 단위로 올림한 시각부터 시작하여 3시간 단위로 7개 출력

# 3. 날씨 timeOfDay 시간 매핑

Time period :

Dawn: 06:00~08:59

Morning: 09:00~11:59

Noon: 12:00~13:59

Afternoon: 14:00~17:59

Evening: 18:00~20:59

Night: 21:00~05:59

# 4. 가장 가까운 다음 시간 예보 정책

1. 9시 알려줘 → 오전 9시(오늘 오전 9시 안 지남) → 오후 9시(오늘 오후 9시 안 지남) → 내일 오전 9시(내일 오전 9시 안 지남) 으로 fallback
2. 아침 알려줘 → 기간 범위의 경우에 끝시간이 지난 경우만 내일로 fallback, 안 그러면 5로 중간에 잘리는 시간 예보 정책
3. 중간에 잘리는 시간 예보 정책
    1. 범위로 요청되었고 앞부분/뒷부분이 예보가능한 범위에서 잘리는 경우 가능한 부분만 예보함

# 날씨 멀티턴 지역(Location) 맥락 유지 및 전환 처리 방식

#### [CASE 1] 이전 대화의 지역 맥락 유지

> **문장 규칙:** 특정 지역(호치민 등)을 명시하여 날씨를 물어본 뒤, 다음 대화에서 **지역을 생략하고 "날씨 알려줘"라고만 발화**하는 경우, 시스템은 이전 대화의 지역 맥락을 그대로 유지하여 동일한 지역의 날씨를 제공합니다. (디폴트 지역인 하노이로 리셋되지 않습니다.)
> 
- **지역 생략 시 이전 지역 맥락이 유지되는 경우**
    - **Turn 1:** "**호치민** 날씨 알려줘" → **[호치민]** 예보 제공
    - **Turn 2:** "날씨 알려줘" → 지역이 생략되었으므로 이전 맥락을 유지하여 **[호치민]** 예보 제공

#### [CASE 2] 새로운 지역으로 맥락을 전환

> **문장 규칙:** 이전 대화의 지역(호치민)과 관계없이, 다음 대화에서 **새로운 지역(하노이 등)을 명시**하여 발화하는 경우, 시스템은 이전 지역 맥락을 지우고 새로 입력된 지역으로 즉시 전환하여 날씨를 제공합니다.
> 
- **새로운 지역 명시로 맥락이 전환되는 경우**
    - **Turn 1:** "**호치민** 날씨 알려줘" → **[호치민]** 예보 제공
    - **Turn 2:** "**하노이** 날씨 알려줘" → 새로운 지역이 지정되었으므로 **[하노이]** 예보 제공

# 날씨 멀티턴 시간 맥락 유지 처리 방식

#### [CASE 1] 주차 정보(다음주 등) + 요일 정보(월요일 등) 결합

> **문장 규칙:** 이전 대화에서 '주차(week)'를 언급하고, 다음 대화에서 주차 없이 '특정 요일'만 말하는 경우, 시스템은 [이전 주차 + 현재 요일]로 맥락을 결합하여 인식합니다. 반대의 경우(요일 선언 후 주차 선언)도 동일하게 결합됩니다.
> 
- **케이스 A: [주차] 선언 후 [요일] 추가**
    - **Turn 1:** "**다음주** 날씨 알려줘" → **[다음주 전체]** 예보 제공
    - **Turn 2:** "**월요일** 날씨 알려줘" → 앞의 맥락을 유지하여 **[다음주 월요일]** 예보 제공
- **케이스 B: [요일] 선언 후 [주차] 추가**
    - **Turn 1:** "**월요일** 날씨 알려줘" → **[이번주 월요일]** 예보 제공
    - **Turn 2:** "**다음주** 날씨 알려줘" → 앞의 맥락을 유지하여 **[다음주 월요일]** 예보 제공

#### [CASE 2] 주차 정보(다음주 등) + 평일/주말 구분 결합

> **문장 규칙:** 이전 대화에서 '평일/주말'을 언급하고, 다음 대화에서 '주차(week)'만 말하는 경우, 시스템은 [이전 평일/주말 + 현재 주차]로 맥락을 결합하여 인식합니다. 반대의 경우도 동일합니다.
> 
- **케이스 A: [평일/주말] 선언 후 [주차] 추가**
    - **Turn 1:** "**평일** 날씨 알려줘" → **[이번주 평일]** 예보 제공
    - **Turn 2:** "**다음주** 날씨 알려줘" → 앞의 맥락을 유지하여 **[다음주 평일]** 예보 제공
- **케이스 B: [주차] 선언 후 [평일/주말] 추가**
    - **Turn 1:** "**다음주** 날씨 알려줘" → **[다음주 전체]** 예보 제공
    - **Turn 2:** "**주말** 날씨 알려줘" → 앞의 맥락을 유지하여 **[다음주 주말]** 예보 제공

# 날씨 멀티턴 항목(Metric) 맥락 유지 및 초기화 처리 방식

#### [CASE 1] 이전 날씨 항목이 초기화되는 규칙

> **문장 규칙:** 특정 항목(습도, 공기질 등)을 물어본 뒤, 다음 대화에서 **"날씨" 같은 일반(General) 발화**를 하거나 "온도/기온"을 물어보는 경우, 이전에 묻고 답했던 특정 항목 맥락은 **완전히 초기화**되며 디폴트(기본) 응답으로 복귀합니다.
> 
- **규칙 1-1: "날씨"(일반 발화)로 인해 초기화되는 경우**
    - **Turn 1:** "하노이 **습도** 알려줘" → 하노이 **[습도]** 예보 제공
    - **Turn 2:** "하노이 **날씨** 알려줘" → 이전 습도 맥락은 리셋되며, 종합적인 **[일반 날씨]** 제공 (화면 카드는 기본값인 공기질(airQuality) 노출)
- **규칙 1-2: "온도/기온" 발화로 인해 초기화되는 경우**
    - **Turn 1:** "하노이 **습도** 알려줘" → 하노이 **[습도]** 예보 제공
    - **Turn 2:** "하노이 **온도** 알려줘" → 이전 습도 맥락은 리셋되며, **[온도 중심의 일반 날씨]** 제공 (화면 카드는 기본값인 공기질(airQuality) 노출)

#### [CASE 2] 이전 날씨 항목이 유지되는 규칙

> **문장 규칙:** [CASE 1]의 초기화 트리거에 해당하지 않는 모든 질의, 즉 항목을 명시하지 않고 대화의 맥락만 이어가는 경우(조사 생략형 등)에는 이전 대화에서 언급했던 특정 항목(습도 등)이 그대로 유지됩니다.
> 
- **항목 맥락이 그대로 유지되는 경우**
    - **Turn 1:** "하노이 **습도** 알려줘" → 하노이 **[습도]** 예보 제공
    - **Turn 2:** "그럼 **내일은?**" (혹은 "다음주는?") → 질문에 항목이 없으므로 앞의 맥락을 이어받아 하노이 **[내일 습도]** 예보 제공

#### [CASE 3] 새로운 날씨 항목으로 전환되는 규칙

> **문장 규칙:** 이전 대화에서 특정 날씨 항목을 요청했더라도, 다음 대화에서 새로운 날씨 항목을 명시하면 이전 항목 맥락은 초기화되고 현재 발화에서 요청한 항목으로 전환됩니다. 이때 지역과 시간 정보는 새로 명시되지 않은 경우 이전 대화의 맥락을 유지합니다.
> 
- **새로운 날씨 항목으로 전환되는 경우**
    - **Turn 1:** “하노이 **습도** 알려줘” → 하노이 **[습도]** 예보 제공
    - **Turn 2:** “그럼 **강수확률**은?” → 지역이 생략되었으므로 하노이 지역 맥락은 유지하고, 날씨 항목은 새로 요청한 **[강수확률]**로 전환하여 하노이 강수확률 예보 제공
